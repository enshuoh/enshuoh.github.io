用VScode寫Markdown

簡介

每個人都有自己習慣使用的編輯器，最近幾年有不少好用的新世代編輯器(Sublime, Atom, VSCode, Bracket…)冒出來。假如你也跟我一樣，喜歡使用VSCode懶得再學其它編輯器，想要只用VSCode一招打天下，這篇文是關於，我自己使用VSCode寫Markdown的一些小分享。

VSCode 內建支援

還記得我剛開始使用VSCode的時候，VSCode對於Markdown的支援還沒有那麼完善，但是現在有很多很棒的東西都已經內建了，實在是太佛拉！

Markdown即時預覽

要打開即時預覽你可以: (以下快捷請依照Mac, Windows或Linux版本自行找尋對應)

  • Cmd + Shift + V
    • 開啟一個顯示預覽的新分頁
  • Cmd + Shift + P
    • 命令列（Command Palette）永遠是你的好朋友 
    • 搜尋Markdown相關指令
  • 通常在你開啟.md檔後，右上角會出現一個開啟側邊預覽分頁的小按鈕

預覽的視窗預設會跟著你的編輯視窗滾動，假如不喜歡的話可以在設定中更改

"markdown.preview.scrollPreviewWithEditorSelection": false,
"markdown.preview.scrollEditorWithPreview": false

擴充套件

工欲善其事，必先利其器。VSCode的擴充套件絕對是它吸引人的一大原因，當你要使用VSCode做某件事時，通常99.9999%有人已經寫好了相關的擴充套件，幫助你省下不少時間拿來多寫幾篇文滑幾篇FB動態。

投資有賺有陪，但是老師在這裡跟你講，在做事前，投資點時間找些好用的擴充套件，絕對是有賺不賠，此外，除了省時以外，當自己的編輯器看起來變得比較酷炫，心情就是好！記得那天早上，光是裝了些擴充套件後，即使一行Markdown都還沒打，我就已經覺得自己充滿了力量(?)

Markdownlint

假如你平常有在寫程式，一定對於lint不陌生，MarkdownLint就是給Markdown用的lint(?)，白話講就是幫你檢查有無Markdown的文法錯誤或是誤用，幫助你維護自己的Markdown文檔整潔。

Markdown Shortcuts

懶人必備快捷鍵擴充，假如你跟作者一樣金魚腦，永遠記不起來Markdown的某些格式，除了快捷鍵外，作者也把它當作cheat sheet在用被毆飛。 這裡我只列出幾個常用的選項

  • 在命令列列出所有支援的選項 Ctrl + M, Ctrl + M
  • 超連結 Ctrl + L
  • 圖片連結 Ctrl + Shift + L

Markdown Preview Github Styling

這個套件的用途是，把原本的預設預覽輸出格式，調成Github的格式，這功能可能不是對大家都有用，由於我自己有在寫Gitbook，或是寫一些要放在Github上的文件，所以可以直接看到放出來大致會長成怎樣還蠻方便的。 
